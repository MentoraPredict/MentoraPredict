# prediction-service

NestJS service for AI-backed academic predictions and recommendations.

> This service merges the former `recommendation-service`. There is no
> Python/local-model pipeline — predictions are generated by calling the
> OpenAI API with deterministic risk data already computed by
> `analytics-service`.

## Responsibility

1. Fetch the deterministic risk snapshot for a student/period from
   `analytics-service` (RF-018 logic — never recomputed here).
2. Fetch the student's current grades per subject from `academic-service`.
3. Send both to OpenAI to get a natural-language `summary` plus a structured
   `recommendations[]` plan.
4. Log every prediction to MongoDB (`prediction_logs`) for history/audit.

## Endpoints

- `GET /api/v1/prediction/students/:studentId/periods/:periodId`
- `GET /api/v1/prediction/students/:studentId/history?limit=`

Both require a user JWT (Bearer), validated the same way as the other
services behind Kong.

## Environment variables

| Variable | Description |
|---|---|
| `OPENAI_API_KEY` | OpenAI API key. If missing, falls back to a deterministic, data-grounded recommendation instead of failing the request. |
| `OPENAI_MODEL` | Defaults to `gpt-4o-mini`. |
| `ANALYTICS_SERVICE_URL` | Defaults to `http://analytics-service:3004`. |
| `ACADEMIC_SERVICE_URL` | Defaults to `http://academic-service:3003`. |
| `MONGO_*` | Standard Mongo connection vars, same convention as analytics-service. |
| `JWT_PRIVATE_KEY` / `JWT_PUBLIC_KEY` | RS256 keys, same shared pair used across the platform. |

## Port

`3006` (the original `prediction-service` port).
